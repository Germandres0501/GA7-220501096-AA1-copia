Principales sistemas y plataformas para gestionar el código fuente

Sistemas de Control de Versiones

Las herramientas de control de versiones permiten a los desarrolladores rastrear y gestionar cambios en el código fuente a lo largo del tiempo. Son esenciales para el desarrollo colaborativo y profesional de software.

Git

Git

Distribuido

Sistema de control de versiones distribuido diseñado para manejar desde pequeños a grandes proyectos con velocidad y eficiencia.

Rápido Distribuido Branching Open Source

Características destacadas:

  • Sistema completamente distribuido
  • Branching y merging eficientes
  • Historial completo offline
  • Integridad criptográfica del código
  • Área de preparación (staging area)
# Inicializar repositorio
git init

# Clonar repositorio
git clone https://github.com/usuario/repo.git
Windows macOS Linux
Creado por Linus Torvalds en 2005
SVN

Subversion (SVN)

Centralizado

Sistema de control de versiones centralizado diseñado para reemplazar a CVS con un enfoque en la simplicidad y estabilidad.

Centralizado Estable Open Source Apache

Características destacadas:

  • Commits atómicos
  • Control de versiones para directorios
  • Historial centralizado
  • Control de acceso detallado
  • Bajo consumo de recursos
# Checkout de repositorio
svn checkout https://ejemplo.com/svn/repo

# Commit de cambios
svn commit -m "Mensaje de commit"
Windows macOS Linux
Desarrollado por CollabNet, ahora parte de Apache Software Foundation
Hg

Mercurial

Distribuido

Sistema de control de versiones distribuido enfocado en el alto rendimiento y la escalabilidad.

Distribuido Escalable Python Open Source

Características destacadas:

  • Interfaz simple y consistente
  • Completamente distribuido
  • Gestión avanzada de branches
  • Extensiones y plugins
  • Alto rendimiento en archivos grandes
# Inicializar repositorio
hg init

# Clonar repositorio
hg clone https://ejemplo.com/repo
Windows macOS Linux
Creado por Matt Mackall en 2005
P4

Perforce Helix Core

Centralizado

Sistema de control de versiones de alto rendimiento que destaca en el manejo de archivos grandes y binarios.

Comercial Empresarial Archivos Grandes Escalable

Características destacadas:

  • Gestión de archivos binarios y grandes
  • Modelo de seguridad robusto
  • Alto rendimiento y escalabilidad
  • Integración con múltiples herramientas
  • Streams para gestión de flujos de trabajo
# Sincronizar workspace
p4 sync

# Enviar cambios
p4 submit -d "Descripción del cambio"
Windows macOS Linux
Desarrollado por Perforce Software, Inc.
TFS

Azure DevOps Server

Centralizado/Git

Plataforma de Microsoft para desarrollo colaborativo que incluye control de versiones, seguimiento de proyectos y CI/CD.

Microsoft Empresarial Integrado Git/TFVC

Características destacadas:

  • Soporte tanto para Git como TFVC
  • Integración con Visual Studio
  • Gestión de proyectos incorporada
  • Integración continua y despliegue
  • Herramientas de administración avanzadas
# Clonar repo Git de Azure DevOps
git clone https://dev.azure.com/org/proyecto/_git/repo
Windows Web
Desarrollado por Microsoft, anteriormente conocido como TFS
F

Fossil

Distribuido

Sistema de control de versiones distribuido con servidor web integrado, seguimiento de problemas y wiki.

Distribuido Todo-en-uno Autocontenido Open Source

Características destacadas:

  • Archivo único autocontenido
  • Servidor web integrado
  • Wiki y sistema de tickets incluidos
  • Sincronización bidireccional
  • Autenticación y permisos incorporados
# Clonar repositorio
fossil clone https://example.org/repo.fossil

# Abrir repositorio
fossil open repo.fossil
Windows macOS Linux
Desarrollado por D. Richard Hipp (creador de SQLite)
CVS

CVS

Centralizado

Sistema de control de versiones centralizado clásico, predecesor de la mayoría de los sistemas modernos.

Veterano Centralizado Open Source Histórico

Características destacadas:

  • Control de versiones a nivel de archivo
  • Trabajo simultáneo en los mismos archivos
  • Operaciones atómicas limitadas
  • Gestión de tags y branches
  • Ampliamente probado y estable
# Checkout de módulo
cvs checkout modulo

# Commit de cambios
cvs commit -m "Mensaje"
Windows macOS Linux
Desarrollado originalmente en 1990, considerado legado
Bzr

Bazaar

Distribuido

Sistema de control de versiones distribuido con enfoque en la facilidad de uso y flexibilidad.

Distribuido Python Canonical Open Source

Características destacadas:

  • Interfaz amigable para usuarios
  • Funciona tanto en modo distribuido como centralizado
  • Integración con múltiples herramientas
  • Soporte para flujos de trabajo flexibles
  • Línea de comandos intuitiva
# Inicializar repositorio
bzr init

# Commit de cambios
bzr commit -m "Mensaje"
Windows macOS Linux
Desarrollado por Canonical Ltd., ahora con mantenimiento limitado

Plataformas de Alojamiento

Estas plataformas proporcionan servicios de alojamiento para repositorios de control de versiones, añadiendo funcionalidades de colaboración, CI/CD y gestión de proyectos.

GH

GitHub

Plataforma Git

La plataforma de desarrollo colaborativo más popular, centrada en Git con características sociales y de colaboración.

Microsoft Social Pull Requests Actions

Características destacadas:

  • Pull Requests y revisión de código
  • GitHub Actions para CI/CD
  • GitHub Pages para sitios estáticos
  • GitHub Copilot para asistencia de IA
  • Gestión de issues y proyectos
Web GitHub Desktop Aplicación móvil
Fundado en 2008, adquirido por Microsoft en 2018
GL

GitLab

Plataforma Git

Plataforma completa de DevOps que cubre todo el ciclo de vida del desarrollo de software.

DevOps CI/CD Todo-en-uno Open Source

Características destacadas:

  • Plataforma DevOps completa
  • CI/CD integrada y potente
  • Disponible en versión autoalojada
  • Métricas y análisis de código
  • Gestión de proyectos incluida
Web Autoalojado
Fundado en 2011, disponible como SaaS y autoalojado
BB

Bitbucket

Plataforma Git/Mercurial

Plataforma de alojamiento de código orientada a equipos profesionales, parte del ecosistema Atlassian.

Atlassian Jira Pipelines Empresarial

Características destacadas:

  • Integración con Jira y herramientas Atlassian
  • Bitbucket Pipelines para CI/CD
  • Pull Requests y revisión de código
  • Disponible en la nube y Data Center
  • Control de acceso granular
Web Data Center
Fundado en 2008, adquirido por Atlassian en 2010
AZ

Azure DevOps Services

Plataforma Git/TFVC

Conjunto de servicios de desarrollo en la nube de Microsoft que incluye control de versiones, CI/CD y gestión de proyectos.

Microsoft Azure DevOps Empresarial

Características destacadas:

  • Repos (Git y TFVC)
  • Pipelines (Build y Release)
  • Boards (gestión ágil)
  • Test Plans (pruebas manuales y automatizadas)
  • Artifacts (gestión de paquetes)
Web Visual Studio
Evolución del antiguo Visual Studio Team Services (VSTS)
